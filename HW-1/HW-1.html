Перелік найчастіших команд GIT:

  • git config --global user.name blabla - команад для задання імені користувача.
  • git config --global user.email blabal@bla - команда для задання емайлу користувача.
  • git init - Ця команда створює у поточному каталозі новий підкаталог з ім'ям .git.
  • git status - Команда відображає стан робочого каталогу та розділу проіндексованих файлів.
  • git add - Команда git add додає зміст робочого каталогу індекс (staging area) для наступного коммiта.
  • git commit - Команда, яка робить (комітить) зміни. Зберігаємо поточні зміни проекту.
  • git log - Команда відображає виконані комміти
  • git checkout - Команда переміщення показника HEAD, також відміняє зміни фала
  • git branch - Команда, яка вміє перераховувати Ваші гліки, створювати нові, видаляти та перейменовувати їх.
  • git push - Команда, яка публікує з локального репозиторія в центральний
  • git pull - Команда, яка загружає і зберігає данні з віддаленого репозиторія в локальний
  • git merge - Команда використовується для злиття однієї або декількох гілок у поточну. Потім вона встановлює вказінвик поточної гілки на результуючий коміт
  • git clone - Команда для створення клону (копії) поточного репозиторія

Що таке .gitignore

.gitignoreb - це файл задопомогою якого приховуються файли і папки від системи контролю версій GIT

Advanced:

  • git revert - Команда для безпечного скасування. Для відкату змін.Команда не видаляє з історії комміти чи батьківські елементи, а створює новий коміт зі скасуванням потрібних дій.
  • git reset - Комнада, яка використовується для скасування локальних змін у репозиторії Git.
  • git rebase - Команда, яка використовує один із двох способів об'єднати зміни, зроблені в одній гілці, з іншою гілкою
  • git cherry-pick- Команда, яка бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці. Ця можливість корисна в ситуації, коли потрібно забрати парочку коммітів з іншої гілки, а не зливати гілку цілком з усіма змінами, що внесені до неї.
  • git stash - Команда для того, щоб помістити поточні зміни, які ви здійснили у файлах, в окреме сховище, та повернути файли до вихідного стану
